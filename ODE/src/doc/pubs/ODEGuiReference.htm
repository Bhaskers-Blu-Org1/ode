    About this document

    This document gives the information necessary for the reader to use the ODE Graphic User Interface.

    See the ODE Documentation Index for a complete list of ODE documents and their purposes.

    Software Requirements

    • The Java JDK 1.1.8 or JRE 1.1.8 or later must be installed in order to run the ODE Graphic User Interface.  Higher versions (1.2 or 1.3) are recommended if unusual behavior occurs with JDK 1.1.
    • The ODE tools.jar file (%release_name%_bnum_tools.jar, where num is the build number) is required and must be added to the CLASSPATH environment variable.
    • The Java Swing library swing.jar is required for JDK1.1, and must be added to the CLASSPATH environment variable.  It is available as part of the JFC package here: http://java.sun.com/products/jfc/download-103.html.  The Swing class library is bundled with JDK 1.2 and higher as part of the default runtime class library (e.g., in jre/lib/rt.jar), so if you have a current release of the Java SDK or JRE, you should not need to download anything further.
    • If you use Exceed as your X server, version 6.2 or higher is recommended; older versions have problems on certain platforms.
    • On AIX, due to security restrictions with xterm, you will need to create a shell script called odeguirun with read/execute permission and put it in a directory that is in your PATH environment variable.  This script is needed to reset LIBPATH to the ODE installation directory, since xterm will clear the contents of that variable upon startup.  Below is a sample of the contents of this script, but you can customize it as desired.
        • #!/bin/ksh
        # This script needs to do a minimum of two things:
        # 1. set the LIBPATH environment variable
        # 2. run the command passed to it, which may contain shell characters
        #    (so it must be passed to a new shell invocation and quoted)
        export LIBPATH=/ode/aix
        /bin/ksh -c "$*"

    Starting the ODE Graphic User Interface

    The ODE Graphic User Interface is a graphical front end to a number of ODE commands, including mkbb, mksb, build, workon, and sbls which are described in the ODE Build Reference. The ODE Graphic User Interface command displays a Sandbox List window containing information about sandboxes based on the contents of an rcfile. The sandboxes are selectable as operands for various operations including: create, delete, workon, build, and show sandbox files. Information about sandbox files is displayed in a Sandbox Files window. The files may be selected for editing.

    Syntax

    java com.ibm.ode.gui [-rc rcfile] [-ed editor] [ODE-options]

    ODE options:  [-quiet | -normal | -verbose | -debug] [-usage | -version | -rev] [-info] [-auto]
     

    Parameters

    -rc rcfile
    Causes the Graphic User Interface to use the file named rcfile to obtain information about the sandboxes. If this option is not specified, it uses the SANDBOXRC environment variable if set, if not set then it uses the $HOME/.sandboxrc file to obtain this information.
    -ed editor
    An editor to use with files in the Sandbox Files window. If none is specified, a default editor will be provided in the edit dialog box based upon the platform that the Graphic User Interface is running on. The user may change this selection in the edit dialog box.

    ODE Options

    -info
    Provides information on what the Graphical User Interface would do if it were executed. The actual command is not performed.
    -auto
    This option does not affect responses in the dialog boxes that the Graphical User Interface displays. Some dialog windows may have default options already selected with the user having the ability to change them, or options may have a default behavior if no user entry is made.
    -usage
    Displays the correct syntax for this command and exits.
    -quiet, -normal, -verbose, -debug
    Controls the amount of message text you receive. -quiet is the most succinct; you receive only essential information, including all FATAL ERROR messages.  -normal is the default level; it causes display of information on the significant steps the program is taking, and warnings. -normal also provides all messages displayed by -quiet. -verbose causes display of detailed information on the steps the program is taking, and all diagnostic errors. It also provides all messages displayed by -quiet and -normal.  -debug provides the highest level of detail available, which may be helpful if a problem is suspected. Some -debug message text may be displayed in the command window where the Graphic User Interface is started. In most cases errors and warnings are displayed in dialog windows.
    -version, -rev
    Displays information on the ODE version level installed and exits.

    Return Codes

        0            Success
        1            Error
        Other    System errors

    Examples

    java com.ibm.ode.gui

    This command runs the Graphic User Interface with the default rcfile. If the file exists and lists sandboxes, they will be displayed in the Sandbox List window; otherwise an empty window will be shown which will provide options for creating sandboxes in the default rcfile.

    java com.ibm.ode.gui -rc /project/.projsbrc

    This command runs the Graphic User Interface with a non default rcfile/project/.projsbrc. If /project/.projsbrc exists and lists sandboxes, they will be displayed in the Sandbox List window; otherwise an empty window will be shown which will provide options for creating sandboxes in the /project/.projsbrc file.

    Using the Sandbox List window

    This window lists sandbox information in tabular format, one sandbox per row. The information is displayed in several columns. There window has several menus and menu options, which are described below.

    Sandbox List window format

    The columns of the Sandbox List window are described below. The information in the Sandbox, Base, Variable Base, and Default columns also can be seen in the output of mksb -list. The Backing Build and Status information comes from running currentsb -chain for each sandbox. The columns and rows of the window may be manipulated by mouse operations or keyboard operations.
    Sandbox
    This column has the sandbox name.
    Base
    This column has the absolute path of the directory that contains the sandbox.
    Variable Base
    If ODE variables were used to define the sandbox base when the sandbox was created, this column has the base before the variables are expanded. The Base column has the result after variable expansion. If no variables were used, the Variable Base column is blank.
    Backing Build
    If this column is checked, the sandbox is a backing build.
    Default
    If this column is checked, it indicates that the sandbox is the default sandbox.
    Status
    This column indicates whether WARNING or ERROR messages can happen when attempting a workon of the sandbox. The normal status is OK. If the status is ERROR, a workon generally will not succeed. ERROR can indicate that the sandbox is missing or a backing build of the sandbox is missing, among other things. A WARNING status may indicate a minor problem, such as a missing Buildconf or Buildconf.exp file. This will be the normal situation when a backing build is first created, before the user creates such files. A WARNING status should not prevent a workon.

    Sandbox List window menus

    The menus and menu options are described. They may be selected by mouse operations or keyboard operations. Some of the options require you to select one or more sandboxes (rows of the table) using the mouse or keyboard. If there are no sandboxes displayed in the window, some options will be grayed out.

    File menu

    Refresh
    This option refreshes the view of the sandboxes. The information in the Sandbox List window is recreated and the sandboxes are displayed in the order that mksb -list would display them. Some Sandbox List operations (create or delete) will cause an automatic refresh. However, if the user creates or deletes sandboxes in separate command windows, the ODE Graphic User Interface will not detect that changes have occurred. The refresh option is useful to bring the window up to date in this case.
    Exit
    This option terminates the Sandbox List window. If the ODE Graphic User Interface has started independent windows, such as build, workon, or Sandbox Files windows, they will continue to run.

    Sandbox menu

    Create Backing Build...
    This option displays a dialog window for creating a sandbox that will be a backing build. This option is a front end to the mkbb command.
    Create Sandbox...
    This option displays a dialog window for creating a sandbox that will be backed by another sandbox. This option is a front end to the mksb command.
    Workon...
    This option displays a dialog window for starting a workon command for a selected sandbox. If the workon successfully starts, it will be running as an independent process in a command window.
    Build...
    This option displays a dialog window for starting the mk command for a selected sandbox. If the build starts successfully, it will be running as an independent process in a command window in a workon environment. When the build ends, the command window still will be in a workon environment so the user may view the tail of the output and enter more shell commands if desired. The user may choose the directory of the sandbox to build in, and the operands to pass to the mk command.
    Show Files...
    This option displays a dialog window for starting a Sandbox Files window for a selected sandbox. Once the Sandbox Files window starts it will run as an independent process. See the Sandbox Files window description for details.
    Delete
    This option deletes the selected sandboxes. This option is a front end for mkbb -undo and mksb -undo. If multiple sandboxes are selected, they are deleted in the order they are shown in a confirmation dialog.

    Help menu

    Help contents
    This option displays some online help.
    Documentation
    This option displays where documentation is to be found.
    About
    This options displays the version and build of the ODE Graphic User Interface

    Using the Sandbox Files window

    The ODE Graphic User Interface Sandbox Files window lists file information in tabular format, one file per row. Unless otherwise specified, the term "file" can also refer to a directory. The information is displayed in several columns. There window has several menus and menu options, which are described below.

    The Sandbox Files window is started from the Sandbox List window and runs as an independent process. The Sandbox List window uses the sbls command to find the files to display for a selected sandbox. The dialog that starts this window provides several options for the user. The user may specify a subdirectory of the sandbox whose files are to be displayed; the default is the src directory of the sandbox. There is an field for optional file specifications for the sbls command. If no file specification is entered, it is "." by default, the current directory. The current directory in this case is either the src directory or the optional subdirectory if the user supplies it. The user has a choice of whether or not recursion into subdirectories is done. The user has a choice of whether files are displayed from just the sandbox or from the sandbox and all its backing chain sandboxes.

    If the user specified more than one file specification, it is possible to have duplicate files when using sbls. The Sandbox Files window eliminates duplicates. It is possible to give file specifications that refer to files outside of the sandbox.

    Sandbox Files window format

    The columns of the Sandbox Files window are described below. The columns and rows of the window may be manipulated by mouse operations or keyboard operations.
    Base
    The sandbox base, or blank if the file is not in the sandbox. If the base and path are concatenated, the result is the absolute path to the file. Breaking apart the absolute path into base and path allows the user to sort by sandbox or by file path irrespective of what sandbox the files are in.
    Path
    The path of the file from the base including root and suffix. If the file is not in the sandbox, the path is its absolute path.
    Date
    The date and time the file was last modified, in a sortable format.
    Size
    Size of the file in bytes.
    Filename
    Root and suffix of the file, so that it is a separately sortable item.
    Suffix
    Suffix of the file, so it is a separately sortable item.
    Type
    This indicates whether it is a directory, file, or other object in the file system. For example, some objects stored in /dev on UNIX might not be recognized as either a directory or a file.
    Pos
    A number indicating the position on backing chain. If the file is not in any sandbox of the backing chain, the value is 0. If it is in the sandbox itself, the value is 1. If it is a file in the sandbox immediately backing the sandbox, the value is 2, and so on.

    Sandbox Files window menus

    The menus and menu options are described. They may be selected by mouse operations or keyboard operations. Some of the options may require you to select one or more files (rows of the table) using the mouse or keyboard. If there are no files displayed in the window, some options will be grayed out.

    Sandbox menu

    Refresh
    This option refreshes the view of the files. The information in the Sandbox Files window is recreated and the files are displayed in the order that sbls would display them, except that duplicates will not be shown twice. However, if the user creates, edits, or deletes files in separate command windows, the ODE Graphic User Interface will not detect that changes have occurred. The refresh option is useful to bring the window up to date in this case.
    Exit
    This option terminates the Sandbox Files window. If the ODE Graphic User Interface has started independent windows, such as file edit windows, they will continue to run.

    Files menu

    Edit...
    This option starts the editor of your choice in a command window, with selected file(s) if there are any. There is an option to start the edit process in a directory of your choice. The edit will be run in an independent process. The editor may be specified with the Graphic User Interface -ed editor command line option. If the editor is not specified then a default initial value will be provided in the editor field of the dialog box based upon the type of platform the Graphic User Interface is running on. You may modify the editor in the dialog box before starting the edit.

    Help menu

    Help contents
    This option displays some online help.
    Documentation
    This option displays where documentation is to be found.
    About
    This options displays the version and build of the ODE Graphic User Interface

    Mouse operations

    These mouse operations apply to both the Sandbox List and the Sandbox Files windows.
    Column manipulation
    The columns may be rearranged by using a mouse on the column labels with a mouse button pressed. They may also be adjusted in width by moving the separators between the column labels with a button pressed. Movement of a particular separator adjusts the sizes of the column immediately to the left, and all columns to the right. The total width and height of the window may be adjusted by the standard method of the system that the program is running on, usually by manipulating the window frame.
    Sorting
    Sorting by column contents can be done with mouse clicks on the column labels. A click sorts in ascending order, and shift-click sorts in descending order. The Refresh menu option can be used to restore the original order.
    Row selection
    A single row may be selected with a left button click. To select or deselect nonadjacent lines press the control key while clicking with the left button.

    Keyboard Operations

    If a Sandbox List or Sandbox Files window has keyboard focus then the following operations are available.
    Menu keys
    The alt key combined with certain other keys can select any menu and certain specific menu items. For example alt-h selects the help menu, alt-x selects the exit menu item, and alt-r selects the refresh menu item. If a menu has focus, there is a key that can be used to select each menu item. An underlined character indicates which key may be used to select a menu item. If a menu item can be selected at any time when combined with the alt key, that information is shown in the menu. The arrow keys may also be used to move focus among the menus and menu items. If a menu item has focus, the enter key can be used to activate the selected action.
    Selections
    The up and down arrow keys will change the selection. Adjacent rows may be selected by combining the shift and arrow keys.

    Environment variables used

     
    CONTEXT This variable designates the machine for which a build would be intended. If a machine name is specified in the build dialog window (-m option), else if the MACHINE environment variable is already set, then CONTEXT is set to that machine name (for example x86_nt_4, rios_aix_4, etc.). If nothing has been specified then the default value is the name of the machine that the tool is executing on.
    MACHINE This always holds name of the machine that the tool is executing on (for example, rios_aix_4, x86_os2_4, etc.). It may be set by the user although it is normally not necessary or recommended.
    SANDBOXRC If set, the specified file is used instead of default .sandboxrc file.
    BUILD_LIST If set, this specifies the build_list file that has a list of all the backing builds.
    ODERCFNAME Control files that define a sandbox (sb.conf) are stored in a directory within the sandbox. If the environment variable ODERCFNAME is defined then the ODE commands use its value for the directory; otherwise, the default directory is rc_files.
    ODESRCNAME If the environment variable ODESRCNAME is defined then the ODE commands use its value for the directory within the sandbox where source files are stored; otherwise, the default directory is src.
