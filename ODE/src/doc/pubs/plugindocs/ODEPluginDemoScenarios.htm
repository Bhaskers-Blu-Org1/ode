ODE Plug-in Demo Scenarios

Target users for the following scenarios(1,2,3,4):

  • Current users of ODE that are planning to use or using Eclipse/WSWB for their development.
  • Users who want to use ODE for nightly/release builds and for packaging, but developers want to use Eclipse/WSWB for code development and local builds.

Scenario 1 (Setup & Configure)

Pre conditions:
  • ODE 4.0 is installed on the client machine. The location of the ODE tools is added to the PATH environment variable.
  • WSWB 2.0 or higher version is installed on the client machine
Actions:
  • Install the ODE Plug-in
    • Select the Help > Software Updates > Update Manager menu item
    • In the Install/Update perspective
    • Add the ODE plug-in site by selecting New > Site Bookmark context menu item of the Feature Updates view. The URL for the ODE Plugin Install/Update site is http://w3.sdwb.ibm.com/plugins/sdwb/install
    • Select the ODE Plugin 4.8.20021120 feature and click the Install button in the Preview view. This opens a wizard that is self descriptive.
  • Configure the ODE Plug-in
    • Uncheck the Perform build automatically on resource modification check box in the Workbench Preferences window.
    • Specify the path to the bin directory of ODE installation in the ODE Tools Directory text field of the ODE Preferences window.
Post conditions:
  • ODE Plugin prototype is installed and configured.

Scenario 2 (Generate Makefiles)

Assumption:

The project contains entire code for a product.

Pre conditions:

  • Scenario 1 is completed
  • A Java/C/C++ project is created
  • In case of a Java project
    • The Use source folders contained in the project option is selected, a source directory is defined and the Build output folder is set for the above java project.
      • To do this open Properties window for the java project, select Java Build Path category, select Source tab, select Use source folders contained in the project option, click Create New Folder button to specify "src" as the source folder and optionally enter "<project name>/export/classes" in the Build output folder text field.
    • A package is created; Java source files are created in the package and are ready for compilation.
  • If a backing build does not exist, then a backing build is created at the location specified in the Backing Build Path field by clicking Create Backing Build button in the ODE page of Properties window.
  • If a backing build exists then the Backing Build Path field in the ODE page of properties window is set to the root directory of the backing build.
  • Sandbox environment is created for the project by clicking Create Sandbox button in the ODE page of Properties window.
Actions:
  • Select the makefile targets that need to be generated in the Makefile targets to generate group box in the ODE page of Properties window.
  • In the Package Explorer or Navigator view, select the project, select ODE > Generate Makefiles from the context menu.
Post conditions:
  • The makefiles are created for each directory
  • Each makefile contains one or more targets selected in the ODE page of Properties window.

Scenario 3 (Build a Project)

Assumption:

The project contains entire code for a product.

Pre conditions:

  • Scenario 2 is completed
Actions:
  • In the Package Explorer or Navigator view, select the project, select ODE > Build Project from the context menu.
Post conditions:
  • A sandbox is created and the project is built

Scenario 4 (Using Existing ODE Sandbox)

Pre conditions:
  • Scenario 1 is completed
Actions:
  • Create a Java project based on an existing sandbox
  • Select File > New > Project menu item
  • Select Java > Java Project in the New Project wizard, click Next button
  • Enter a project name, uncheck the Use default check box and enter the root directory of the existing sandbox, click Next button
  • Click Yes on the message that appears, this creates a project
  • In the Source tab, select Use source folders contained in the project option, click Add Existing Folders button to specify "src" as the source folder and enter "<project name>/export/classes" in the Build output folder text field.
  • Specify the location of the sandbox RC file and backing build in the ODE project properties window.
  • In the Package Explorer or Navigator view, select the project, select ODE > Build Project from the context menu.
Post conditions:
  • The project is built.

Limitations on the source folder:

  • MUST be only one source folder per project (WSWB allows you to have many)
  • MUST be explicitly given (cannot use the project itself as the source folder)
  • MUST be a single directory under the project (i.e., a source folder like "foo/bar/src" is not allowed)
  • MUST be called "src"
