ODE Style Guide

Highlighting Conventions

The style conventions used in ODE command syntax are described in How to Read Syntax Statements. The following highlighting conventions are also used in this book:

 

Bold
Actions, backing build areas, commands, directories, drive names, execs, flags, filenames, field names, machine names, names of sandboxes, parameters, passes, targets, WARNING!, Note:, and Caution appear in bold. Valid abbreviations for commands are also in bold.
Italic
Arguments, options and variables whose names or values must be supplied by you appear in italics. Italics are also used for emphasis and for titles of books.
Monotype
Examples of specific data values, examples of expected output, messages, or information that you should type appear in monotype.

 

How to Read Syntax Statements

Style Usage
bold Items you must enter exactly as shown, such as commands, flags, and symbols. The valid abbreviations for commands and flags appear in bold, with the optional part of the command or flag shown in regular type. For example, workon or mk.
Italic Arguments or options whose values you must supply. For example, sandbox_name or text.
... Parameter can be repeated on the command line. For example, -machine2... means that you can enter more than one argument for machine_names.
[ ] Optional parameters are enclosed in square brackets. For example, [-dir sandbox_dir].
{ } There is more than one parameter choice, but one is required.
| Choose one parameter only. [a|b] indicates that you can choose a or b, or neither a nor b. {a|b} indicates that you must choose either a or b.

Notes

  • There should be one space between sentences.
  • Paragraphs should not be indented.
  • The "default" font on an editor should be used so the font adjusts to selected browsers.
  • The page color or background should be white (#FFFFFF) which is the default on most editors.
  • Acronyms should be spelled out at the first occurrence in a document.
  • Use bold type when referring to a specific option or variable. For example: the -I option, the playa1 machine name.
  • Terms in a definition list should be bold.
  • Links should have consistent bolding. For example:"mk command" instead of "mk command".
  • Numbers in a list (ordered lists) are used only when sequence is important. Otherwise, use an unordered list.
  • Titles of books used as links should be italicized.
  • ODE documentation links should specify ODE xxxxx and be italicized. For example, ODE Build Reference.
  • Font size should be 12pt. variable width for normal text and 10 pt. fixed width for example text.
  • Use bold type for "Note:", "Warning:", or "Caution:". For example, Note: Most real-world project backing builds are populated with source code extracted from a library.
